# 02 — Privacy Filter — Requirements

## Overview

Prevent sensitive clipboard content (passwords, one-time tokens, transient pastes) from
ever being captured or stored, by honoring the de-facto `org.nspasteboard.*` type hints
that password managers and privacy-aware apps set.

## Background

There is a community convention ([nspasteboard.org](http://nspasteboard.org)) for marking
clipboard content that history managers should ignore:

- `org.nspasteboard.ConcealedType` — sensitive (e.g. a password). Never record.
- `org.nspasteboard.TransientType` — short-lived; should not be stored in history.
- `org.nspasteboard.AutoGeneratedType` — generated by automation, not user intent.

Password managers (1Password, etc.) set `org.nspasteboard.ConcealedType` when they copy
a password. Honoring it is the core privacy guarantee of the app (NFR-2).

## User stories

- **US-1.** As a user, I want passwords I copy from my password manager to never appear in
  clipboard history.
- **US-2.** As a privacy-conscious user, I want transient/auto-generated clipboard
  content excluded from a persistent record.

## Acceptance criteria (EARS)

- **02-AC-1.** THE SYSTEM SHALL maintain a set of skip types:
  `org.nspasteboard.ConcealedType`, `org.nspasteboard.TransientType`,
  `org.nspasteboard.AutoGeneratedType`.
- **02-AC-2.** WHEN a pasteboard entry's available types intersect the skip set, THE
  SYSTEM SHALL skip it: no `ClipItem` is captured, stored, or persisted.
- **02-AC-3.** IF a pasteboard entry's types do not intersect the skip set, THEN THE
  SYSTEM SHALL allow normal capture to proceed.
- **02-AC-4.** THE SYSTEM SHALL apply the filter using the raw type identifiers obtained
  from `PasteboardReading.availableTypes()` (before reading content), so concealed
  content is never even read into memory as a `ClipItem`.
- **02-AC-5.** THE SYSTEM SHALL expose the filter as pure, side-effect-free logic
  (`shouldSkip(types:) -> Bool`) that is unit-testable without a pasteboard.

## v0.1 (Latch) addendum

- **02-AC-6.** The "Ignore passwords" Settings toggle (feature 09) SHALL enable/disable the
  concealed-type skip; default ON. (Transient/auto-generated are always skipped.)
- **02-AC-7.** Incognito pause (feature 13) is a separate, broader control that stops all
  capture regardless of type.

## Out of scope

- User-configurable allow/deny lists or per-app rules (post-v0.1).
- Heuristic detection of secrets by content (we only honor explicit type hints).
